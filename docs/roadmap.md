# Roadmap y Cronograma (OKRs)

## Objetivo (Q1-Q2)
Lanzar MVP funcional en entorno de demostración académica en 12 semanas.

## OKRs (ejemplo)
- O: Entregar MVP de SMARTVIAL
  - KR1: Registro digital completo de incidentes (80% de campos) — 4 semanas
  - KR2: Dashboard básico con KPIs — 6 semanas
  - KR3: Portal ciudadano para consulta — 8 semanas

## Cronograma (12 semanas)
- Semana 1-2: Análisis y diseño UX/UI
- Semana 3-4: Arquitectura y BD
- Semana 5-7: Backend y API
- Semana 6-8: Frontend y Mapa SIG
- Semana 9: IA (predictor prototipo)
- Semana 10: QA y pruebas de integración
- Semana 11: DevOps y despliegue demo
- Semana 12: Documentación final y demo

## Entregables por iteración
- Iteración 1 (S1-2): Docs, wireframes, design system
- Iteración 2 (S3-5): API básica + DB
- Iteración 3 (S6-8): Frontend y mapa
- Iteración 4 (S9-12): IA, QA, despliegue

Ver `docs/analysis.md` y `docs/test-cases.md` para detalles de aceptación.
